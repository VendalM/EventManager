# Менеджер событий (EventManager)

Простое и понятное API для управления вашими событиями. Создавайте, редактируйте и удаляйте события с помощью удобных HTTP-запросов.

## Что умеет это API?

API поддерживает все базовые операции (CRUD):
*   **Посмотреть все события** — `GET /api/event`
*   **Найти событие по номеру** — `GET /api/event/{id}`
*   **Добавить новое событие** — `POST /api/event`
*   **Исправить событие** — `PUT /api/event/{id}`
*   **Удалить событие** — `DELETE /api/event/{id}`

## Как это устроено внутри?

Проект разделен на логические части, чтобы код был чистым и понятным:
*   **Модели (`EventManager.Models`)** — описывают, как выглядит событие.
*   **Приложение (`EventManager.Application`)** — содержит всю умную логику по работе с событиями.
*   **Презентация (`EventManager.Presentation`)** — отвечает за обработку запросов и показ документации Swagger.

Сервисы подключаются автоматически, а документация генерируется на лету.

## Запускаем за минуту

1.  Проверьте, что у вас установлен **.NET 8 SDK**. Если нет — скачайте с официального сайта Microsoft.
2.  Скачайте проект.
3.  Зайдите в папку с проектом: cd EventManager/EventManager.
4.  Запустите приложение: dotnet run.
5.  Теперь открывайте браузер по адресу http://localhost:5000/swagger (порт может отличаться — посмотрите в выводе консоли после запуска) 
и пробуйте отправлять запросы. Удачного тестирования! 🚀
